# 毎朝Routine用プロンプト

game-studio-daily Skill が利用可能な場合は、そのSkillを使用して毎日のゲーム開発Routineを実行してください。

もしこのセッションで game-studio-daily Skill が見えない、または呼び出せない場合は、以下の内容を game-studio-daily Skill と同等の指示として扱い、そのまま実行してください。

対象リポジトリの現在の状態、Issue、PR、直近の差分を確認し、今日中に「プレイヤーが体感できる小さな遊び」を1つ完成させるための開発方針を日本語で整理してください。

必ず守ること：
- ログ、Routine報告、QA結果、リファクタ報告、Issue要約、PR要約、意思決定記録は必ず日本語で出力する。
- コード、ファイルパス、コマンド名、API名、モデル名、ブランチ名、コミットハッシュは英語のままでよい。
- 大きな機能追加ではなく、今日触って分かる小さな変化を優先する。
- 他ゲームの参考要素は、そのままコピーせず、面白さの構造だけ抽出する。
- 勝手に大規模リファクタリングしない。
- 機能追加PRとリファクタリングPRは分ける。
- 本番DB変更、本番環境変更、破壊的変更、削除は行わず、必要な場合は提案に留める。

出力：
1. 現在の状況
2. 今日作るプレイヤー体験
3. プレイヤー代表の要望
4. ディレクター判断
5. 今日の最小仕様
6. 必要な作業
7. Issue案
8. QA観点
9. リファクタリング候補
10. 今日の完成判定
11. 最初に着手する1手
