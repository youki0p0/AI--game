# AI--game

Claude Code による小規模ゲーム開発プロジェクト。

## 目的

毎日、プレイヤーが体感できる小さな遊びを1つ完成させる。

## 利用可能なスキル

### `/game-studio-daily`

毎日のゲーム開発ルーティンを実行するスキル。

```text
/game-studio-daily          → 今日の開発方針を整理（daily モード）
/game-studio-daily daily    → 同上
/game-studio-daily review   → 直近2日レビュー
/game-studio-daily weekly   → 週次ディレクター総評
/game-studio-daily monthly  → 月次ロードマップレビュー
/game-studio-daily refactor → リファクタリング候補整理
/game-studio-daily graphics → グラフィック素材パイプライン
/game-studio-daily qa       → QAチェック観点
```

スキル定義: `.claude/skills/game-studio-daily.md`

## ワーキングアグリーメント

- 毎日、プレイヤーが体感できる小さな遊びを1つ完成させる
- 大きな機能追加ではなく、触って分かる変化を優先する
- 1 Issue = 1 PR を基本にする
- 機能追加 PR とリファクタリング PR は分ける
- 勝手に大規模リファクタリングしない
- 他ゲームの参考要素は、そのままコピーせず、面白さの構造だけ抽出する
- ログ・報告・QA・Issue/PR 要約・意思決定記録は必ず日本語で出力する
- 本番 DB 変更、本番環境変更、破壊的変更、削除は行わず、必要な場合は提案に留める

## 完成判定（Definition of Done）

毎日の差分は以下をすべて満たしたら完了：

1. プレイヤーが何をすればいいか分かる
2. プレイヤーが何かを選べる、または操作できる
3. 選択や操作の結果が分かる
4. 成功・失敗・報酬・成長・発見・演出のいずれかがある
5. 次にやりたいことが残る
6. スマホで最低限操作できる
7. バグで進行不能にならない
8. 今日の差分として説明できる
