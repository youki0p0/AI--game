## デッキ構築の統計最適化（Sonnet5並列探索＋決定的A/B, 追記）

- 重要: エンジンにRNGシード無し。同一デッキでも OVERALL が ±0.05 ブレる（真の base ≈ 0.58, 以前の0.64は上振れ）。
- 最良候補(Morpeko/EnergySearch/Recycler → Boss+1/Xerosic 2→4): OVERALL 0.597 vs base 0.581。

- 結論: **デッキ微調整では誤差程度しか動かず、base Grimmsnarl がこのパイロットでほぼ最適**。
  伸びしろはデッキ構築ではなく **パイロット(操縦)** にある。
- 確定知見: Spikemuth Gym×4=MegaStarmie対策に必須 / Boss's Orders増量=逆効果 / 安定ドロー厚めが有効。

## パイロット(操縦)最適化: 探索深さ＋勝ちパターン発掘（追記, Archaludon採用後）
「伸びしろはパイロット」の続き。simple_bot(固定スコア)を **全合法手探索＋静的評価(state_eval)** に置換し、
深さ・パターンを実測最適化した記録。相手はsimple_bot操縦の実メタ、先後入替。

### 1. パイロット置換: 固定スコア → 探索
- パイロット比較(N=50/相手・計250戦): 探索 0.752[0.698,0.806] > simple_bot 0.656[0.597,0.715]（+0.10）。
  接戦3柱が全改善(vs Grimmsnarl 0.58→0.72 / Alakazam 0.46→0.70 / MegaStarmie 0.48→0.58)。

### 2. 探索深さの最適化: 奇数=強い/偶数=弱い、かつ深さ3が最良
- 深さ1〜4(N=12): 1:0.56 / 2:0.44 / 3:0.75 / 4:0.42 →「**奇数=強い/偶数=弱い**」。
  理由: 奇数深さは *自分のターン終了時(実盤面)* で評価。偶数深さは *相手ターン終了時* で評価するが、
  相手の隠れ手札/山は自デッキから再構成した"空想"なので誤差が乗る。
- 深さ1 vs 3(N=60・計180戦): 深さ3 0.739[0.675,0.803] > 深さ1 0.633[0.563,0.704]（+0.11, 全マッチ改善）。
- 深さ3 vs 5(N=48・計288戦): 深さ3 0.708[0.634,0.783] > **深さ5 0.597[0.517,0.677]**（深さ5は悪化）。
  理由: 深さ5は相手ターンを*2回*模擬 → 空想相手の誤差が複利で増幅し、先読みの利得を上回る。
- 確定: **深さ3(自分→相手→自分)が最良**。1=浅すぎ(反撃を見ない)/5=深すぎ(空想誤差)。採用。約2.4s/戦。

### 3. 勝ちパターン発掘（567戦, turn≈自分3ターン目の盤面特徴 × 勝敗の lift）
- 相手アクティブをKO圏内 **+0.227** / サイド先行 +0.224 / 自アクティブ エネ≥3 **+0.190** /
  ブリジュラスが自アクティブ +0.156 / 進化済 +0.130 / **自分が被KO圏内 −0.243**。
- 距離パターン: 「序盤にブリジュラスを*アクティブ*へ据えエネ3を積み、KO圏を作りつつ被KO圏を避ける」。

### 4. 勝ちパターンの明示的コード化(オープニングブック)は無効 → 探索が既に実現
- turn≤6 に「ブリジュラス=アクティブ+エネ3」を加点する opening_bonus を state_eval に追加しA/B(深さ3・計288戦):
  base 0.674[0.597,0.750] vs opening 0.646[0.568,0.724] → **改善せず(むしろ微減)**。
- 理由: state_eval が既に エネ充足/KO脅威/被KO/マテリアル を評価しており、発掘パターンは**評価関数に内包済み**。
  深さ3探索がこれらの勝ち筋を*動的に*実現するため、静的な追加ボーナスは二重計上で逆効果になりうる。
- 結論: **発掘パターンは実在するが、現エージェントは探索で既にそれを打っている**。明示ブックは不採用。
  伸びしろは「評価に無い要素(手順・カード固有効果の順序)」の発掘に移る。

### 5. 相手モデル推定（相手型を分類し探索へ正しいデッキを与える）も無効
- 相手の見えるカード(場/トラッシュ/手札枚数)を署名カードで型分類し、探索の*相手側*再構成に
  推定デッキを与える版を実装しA/B。
  - 深さ3(N=48): base 0.694[0.619,0.770] vs oppaware 0.660[0.582,0.737] → **改善せず**。
    理由: 相手の即時脅威(KO)は"見えている"アクティブ＋エネで決まり、隠れデッキに依存しない。
  - 深さ解禁狙い(N=40): base深さ3 0.700 vs oppaware深さ4 0.617 vs oppaware深さ5 0.617
    → **深い探索は依然として悪化**。相手デッキを当てても、(a)隠れた個別ドローは依然当て推量、
    (b)ロールアウトの相手方策が state_eval の粗い貪欲プレイで実相手と異なる、ため深さとともに方策誤差が複利で乗る。
- 結論: **深さ3・ARCHベース再構成が堅牢な最適点**。伸びしろは「深さ/相手デッキ」ではなく、
  もし在るとすれば「ロールアウトの相手*方策*の質」か「学習した評価関数」で、いずれもエンジンの
  ±0.05ノイズ下では費用対効果が不透明。現行の深さ3探索版を確定として維持する。

## 実ラダー敗因分析（ユーザー提供の実戦ログ 4件）
実ラダーの連敗ログ(83103875勝/83104587/83105287/83106731負)を解析。デッキは全て我々のARCH_DECKと完全一致。

### 決定的発見1: 稼働版は旧 simple_bot（探索版が未アップロード）
- 手番所要時間 0.5〜1ms/手（探索版は10〜100ms/手）→ 稼働中の操縦は simple_bot。
- 勝ち試合83103875のエネ付けが「毎ターン アクティブへ、最大7本、控えは0本」= simple_bot の指紋
  （ATTACHを一律1000点→先頭=アクティブを常に選ぶ。付け先の戦略なし）。探索版は控えバックアップを積む。

### 決定的発見2: 連敗の大半は「勝てる/有利な試合を simple_bot が落とした」
- 敗因の相手を特定し、探索版(深さ3)で実測:
  - vs Crustle(ex技無効の壁): **0.275** ← 唯一の真の天敵（構造的不利）。
  - vs Mega Lucario ex(闘, id678): 実測 **0.80**（弱点でも何でもない。simple_botが落としただけ）。
  - vs Mega Abomasnow ex(id723, 弱点=鋼): Archaludonが弱点を突ける **有利マッチ**（simple_botが落とした）。
- Archaludonの弱点は炎(type2)のみ。上記の敗戦相手は闘/水で弱点ではない。

### 結論と打ち手
- **最優先: 探索版(dist/submission.py, 深さ3)をKaggleにアップロード**。稼働中の simple_bot が有利マッチを
  落としているのが連敗の主因。探索版は Lucario 0.80 / Abomasnow 有利 と勝てる。
- 唯一の構造的天敵は **Crustle**(0.275)。これは壁デッキ対策(非ex Cinderace 等)の別課題。
- エネ配分: 探索版は「アクティブがコスト3到達後に控えへ」を概ね実現。余剰エネ価値を下げる調整はA/Bで
  勝率中立(0.660→0.667)かつ「アクティブ未完成で控え先行」の別の偏りを生むため不採用。

### 6. 相手モデルの最強版（推定デッキ×実パイロットrollout）も勝率を"下げる"
ユーザー案「相手デッキを推定し、実パイロットで相手の動きを再現して対策する」を最強の形で実装しA/B(N=30):
- base(汎用探索 深さ3): **0.711**[0.617,0.805]
- oppreal 深さ3(推定デッキを simple_bot で操縦しロールアウト): 0.622
- oppreal 深さ4: 0.567
→ **相手を正確にモデル化するほど悪化**。理由: 特定の相手継続手に賭けると過適合＋分類/rolloutの分散増。
  汎用探索の「自分の結果盤面を評価」は中立的で頑健。相手の即時脅威は"見える盤面"で足りている。
- 分類自体はエネ+ポケモン署名で ~100% 当たるが、**識別はボトルネックではない**（当てても使い道で勝率が落ちる）。
- 結論(確定): **相手モデリングはこのエンジン/評価では逆効果。汎用深さ3探索が最良**。
  対策インデックス(行動模範)が効くのは汎用探索が負ける Crustle 等の構造的天敵のみだが、それはデッキ火力の問題
  (非exは Duraludon 80/ターン ≒ Crustle回復 で削りきれない。Cinderace は進化元が無く戦力化しにくい)。
