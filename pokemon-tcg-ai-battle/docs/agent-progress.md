# PTCG エージェント開発 進捗ログ（vs Buddy 勝率改善）

目標: vs Buddy 勝率 0.65 以上のエージェント。

## 結論（現時点）

**未達。** 多数の手法を実装・検証したが、いずれも vs Buddy 勝率は **約 0.35〜0.50** に留まる。
Buddy はそのデッキ(Fighting/Mega Lucario ex)に高度に最適化された強いルールベースで、
同一デッキでは我々の汎用評価＋浅い探索が実質ミラー（≈0.50）になり、超えられない。

## 試した手法と結果（20〜80戦で測定）

| 手法 | vs Buddy 勝率 | 所見 |
|---|---|---|
| 1手読み(各option→即時局面を評価) | 0.17 | 近視眼。ENDや小攻撃を選びがち |
| ターン内rollout(自ターン終端を評価) | **0.50** | 大幅改善。buddyと互角 |
| 2手番rollout(相手応手も貪欲) | 0.30–0.50 | 相手モデルが不正確で悪化 |
| 学習評価(Texel)を葉に使用 | 0.25 | 位置分類は良いが行動価値として破綻(Goodhart) |
| buddyを探索内の着手方策に使用 | 0.42 | 逸脱後のbuddy再計画が不正確 |
| ガード付き上書き(明確改善時のみ) | 0.40 | 評価が小マージンで誤ランク→逸脱が損 |
| MC終局ロールアウト(1サンプル) | 0.375 | 1サンプルはノイズ過大。多数は重すぎ |
| 勝率目的の重みhill-climbing | 0.43 | 重み調整では壁を越えられない |
| **Psychic弱点デッキ + 汎用探索** | 0.50 | デッキ優位だが操縦不足で活かせず |
| Psychic弱点デッキ + 専用パイロット | 0.35–0.45 | テンポ負け(下記) |

## 重要な発見

1. **探索基盤は完成・動作**: `search_begin/search_step`（隠れ情報を既知デッキから再構成して投入）で
   実局面の先読みが可能。`eval/search_agent.py`。
2. **同一デッキの天井 ≈ 0.50**: `state_eval`-greedy が buddy のヒューリスティックとほぼ同じ手を選ぶため
   実質ミラー。評価の質が律速で、わずかな逸脱はむしろ損。
3. **Python ラッパの `search_step` が約1ms/手**（JSONパース）で、多数シミュレーションのMC/ISMCTSは非現実的。
4. **デッキ優位は実在**: buddy の主力 Mega Lucario ex(HP340) は弱点 PSYCHIC(×2)。
   **Iron Boulder(基本, 1P+1任意で170)** は弱点で 340 = OHKO（3サイド）。
   Iron Boulder(HP140) は Lucario の Aura Jab(130) を耐え、2エネ到達で Lucario を OHKO できる。
5. **デッキ優位が勝率にならない理由＝テンポ**: 手張りはターン1回。我々の攻撃は2〜3エネ要求で、
   起動前に buddy(Aura Jab 130/1エネ で速攻＋エネ加速持ち)にKOされ、サイドをほぼ取れない(平均0.4枚)。
   競技デッキに必要な「エネ加速」と「buddy級の精緻なパイロット」が無いと活かせない。

## 0.65 到達への現実的な道筋（次セッション向け）

A. **競技レベルの Psychic 弱点デッキ + 専用パイロット**（最有望）
   - エネ加速の組み込み（例: Chansey 124 / Terapagos 234 / Mega Audino ex 1006 の0エネ加速技、
     または該当する特性持ち）でテンポ問題を解消。
   - パイロットを buddy 同等に作り込む（セットアップ→Iron Boulder起動→2エネ→弱点OHKO の一貫運用、
     にげ/ベンチ管理/ドロー/サーチの正しい使用）。naive版(`eval/psy_pilot.py`)は要強化。

B. **より深い探索**（ISMCTS）
   - 現状の `search_step` ~1ms/手では多数シミュレーションが重い。高速な状態取得経路が必要。

C. **評価関数の本質的改善**
   - 行動価値として健全な特徴量設計（学習Texelは符号が壊れ greedy で破綻）。

## 追補: 測定ノイズの壁とデッキ路線の限界

- **測定ノイズが致命的**: エンジンRNGがシード不可のため、同一configでも40戦の勝率が
  実行ごとに 0.25〜0.475 と大きく振れる(std大)。0.65 を有意に確認するには config 毎に
  数百戦が必要で、反復チューニングが事実上機能しない。
- **Psychic弱点デッキ + 専用パイロット**: 勝ち筋(Iron Boulder で Lucario OHKO)は理屈上強いが、
  実測は ≈0.25〜0.50。テンポ負け(手張り1回/ターン)とパイロットの未成熟が原因。
  さらに Boss's Orders/Lillie/Hero Cape を入れた強化デッキ(v4)は、複雑効果でエンジンが
  不安定化(無出力クラッシュ/ハング)したため安定版(v3)に差し戻し。
- **結論(確定)**: 現有の手法・この実行環境(Pythonラッパ ~1ms/手, RNGシード不可)では
  vs Buddy 0.65 は未到達。ベストは「ターン内rollout探索エージェント」≈0.50(互角)。
  0.65 には (A)競技デッキ+buddy級専用パイロットの作り込み、(B)高速状態取得→多数ISMCTS、
  (C)行動価値として健全な学習評価、のいずれか(いずれも本セッション規模を超える)が必要。
