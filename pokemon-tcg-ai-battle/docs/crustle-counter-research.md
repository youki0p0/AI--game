# イワパレス(Crustle)対策 WEB調査＆エンジン適用分析

実ポケカ環境の「イワパレス(特性しんぴのいしやど=相手のポケモンexの技ダメージを受けない)」
対策を各所から調査し、CABTエンジンに存在するカードへ対応付けた。

## イワパレスの強さと弱点(要点)

- 強さ: **ex主体デッキの攻撃を完全シャットアウト**。exしか殴り手がいないと詰む。
- 弱点(調査で判明した共通見解):
  1. **非exの攻撃は普通に通る**（特性はexにしか効かない）
  2. **イワパレス自身の火力は低い**（120程度）＝殴り返しでOHKOされない
  3. **単サイド**（倒しても相手は1枚しか取れないが、こちらも非exなら対等）
  4. **弱点は炎(×2)**
  5. HP増強は**ヒーローマント/ゴージャスマント**依存 → スタジアムで無効化できる
- メタ的総括（複数記事の一致見解）:
  **「ハードに対策するより、非exワンパン手段を自然に持つデッキで戦う方が得策」**
  （イワパレスは採用率〜1%のアンチメタ。180点非exワンパンが複数あるデッキには勝てない）

## エンジンで使える具体的勝ち筋（優先度順）

1. **炎非exワンパン軸（本命）**: **Ethan's Typhlosion(354)** の Steam Artillery 160 は
   炎弱点×2で **320ダメージ** → ヒーローマント込み(250)でもCrustleをOHKO。**単サイド**なので
   サイドレースも負けない。+ **Jamming Tower(1246)** でマント剥がし、+ 序盤の安定枚数。
   - 注意: Steam Artillery は3エネ。Buddy Blast(40/1エネ)で繋ぎつつ加速が要る。
     以前の fire_single(0.015) の敗因は「汎用パイロットで弱点OHKOを決められず、
     スパイキーエネの反射＋回復に削り負けた」＝**デッキではなくパイロットの問題**。
     専用パイロットで弱点OHKOを最短で通せば覆る見込み。
2. **特性貫通軸**: **Mega Lopunny ex(849)** Spiky Hopper 160(効果無視)。ただし
   megaEx(3サイド)で、"効果無視"のエンジン実装可否の検証が必須。
3. **非ex雷ワンパン**: **Luxray(1037)** Strong Volt 200。Jamming Tower併用でマント込みも射程。

## 我々のCrustleボット固有の注意

現在の CRUSTLE_DECK は Hero's Cape(1159)×1、ミストエネ・グロウグラス・スパイキーエネを採用。
- **Jamming Tower は Hero's Cape の +100HP を無効化**できる（素150へ）。
- **影縛りの逃げ縛りはミストエネで無効**（ダメージは通るが釘付けにできない）。
- **スパイキーエネの反射20**は、OHKOしてしまえば1回分で済む（延命されなければ無害）。

## 結論

**「非exで弱点(炎)OHKO ＋ Jamming Tower でマント剥がし」** が最有力。
Crustleは低火力・単サイドなので、こちらが非exでワンパンし続ければサイドレースで勝てる。
ドラパルト(ex主体)で正面から割るより、**Ethan's Typhlosion 軸の炎非exデッキ**を新規に組み、
専用パイロットで弱点OHKOを最短で通す方向が、エンジン上でも実現性が高い。

## 【追記 2026-07-01】探索版パイロットでの対Crustle tech 実測（結論: 割り切り推奨）

現行の提出＝探索版(深さ3)で、上記の対策カードを実際に積んで対Crustleを実測した。

### Crustleの仕組み（エンジン再確認）
- Crustle(345) スキル「Mysterious Rock Inn」= 相手の **ex/megaEx** の技ダメージを全無効化。Archaludon ex は **0**。
- HP150(Cape込250)・回復 Jumbo Ice Cream 80 / Cook 70・弱点=炎(2)。

### 探索した対策カード（エンジン実データ）
- **非ex無色クローザー**（既存の鋼エネで撃てる）: Bouffalant233(150/4), Snorlax1072(160/4), Hop's Snorlax304(140/3)。※全て回復とほぼ相殺で削り切れない。
- **炎(弱点2倍)アタッカー**: Victini490(120→240), **Volcanion663(130→260, Cape込OHKO)**, Reshiram794。※炎エネ(id2)スプラッシュ要。
- **妨害**: Judge1213 / Harlequin1223 / Lucian1237 / Eri1186。
- 付随発見: **Cinderace666 は Stage2(evolvesFrom=Raboot)で進化元がデッキに無く場に出せない＝実質4枚の死に札**。

### 実測結果（探索深さ3・先後入替）
| 施策 | 対Crustle | 他マッチ |
|---|---|---|
| ベース | 0.25〜0.375 | — |
| 無色クローザー投入 | 0/6〜0/10（悪化） | — |
| Judge×2〜4 | 0/10〜2/10（効果薄） | — |
| 炎スプラッシュ(Volcanion×2+炎エネ×4) | 0.04 | **MegaStarmie 0.67→0.42 回帰** |
| 教育版eval(ex壁でex技=0と評価) | 0.25(教育なし0.375,N=40) | 有意差なし → **revert済み** |

### 結論
1. **Crustle は我々のデッキ＋汎用探索パイロットに対し構造的天敵**。理論上正しいカードを積んでも勝率が上がらない:
   (a)非ex打点 ≒ 回復で削り切れない / (b)炎スプラッシュは一貫性を薄め**頻出マッチを落とす赤字** / (c)**汎用探索パイロットは"壁専用の勝ち筋"を実行できない**（カードを積んでも使いこなせない）。
2. **eval教育も無効**（代替の勝ち筋が無いため）。提出は実証済みの深さ3版を維持（revert済み）。
3. **推奨**: Crustle は希少な Day-1 stall。専用techは頻出マッチ犠牲の赤字 → **割り切って非対応、デッキは頻出メタ最適のまま維持**が期待勝率的に正しい。
4. **本当に効く"教育"**: 稼働中の旧simple_bot → **探索版(深さ3)への差し替え（dist/submission.py のアップロード）** が最優先。勝てるラダー戦(Lucario/Abomasnow)を実際に拾う実証済みの改善。
